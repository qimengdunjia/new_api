§ Codex CLI · 分系统安装

Codex CLI 安装与配置指南

请先选择你的操作系统和安装方式。推荐使用 OpenAI 官方独立安装器；只有选择 npm 安装时才需要先安装 Node.js。

推荐方式

Windows 使用官方 PowerShell 安装器；macOS/Linux 使用官方 Shell 安装器。两者都不要求 Node.js。

Node.js 是否必须？

不是。仅 npm 路线需要 Node.js/npm；当前 npm 包声明 Node.js 16+，实际建议安装最新 LTS。

官方仓库环境基线

macOS 12+、Ubuntu 20.04+/Debian 10+；Windows 推荐 Windows 11 + WSL2。内存最低 4 GB，建议 8 GB。

01

先选择 安装方式

官方独立安装器（推荐）

直接安装 Codex 原生程序，不依赖 Node.js/npm。适合绝大多数用户，升级时重新运行同一条安装命令即可。

npm 安装

适合已经在使用 Node.js 的开发者。需要 Node.js 16+ 和 npm，建议使用最新 LTS，避免系统仓库中的过旧版本。

Homebrew 安装

适合 macOS 用户。当前官方命令是 brew install --cask codex，不是 brew install codex。

WSL2 安装

适合 Windows 上需要完整 Linux 工具链、容器和 Shell 脚本的用户；进入 WSL 后按照 Linux 步骤安装。

不要同时混用独立安装器、npm 和 Homebrew。多种安装共存容易出现“升级了但终端仍运行旧版本”的 PATH 冲突。

02

Windows 安装步骤

方案 A：官方 PowerShell 安装器（简单、无需 Node.js）

打开 PowerShell，执行 OpenAI 官方快速开始提供的安装命令：

powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"

安装完成后关闭并重新打开 PowerShell，然后验证：

codex --version Get-Command codex

方案 B：WSL2（推荐给开发环境用户）

OpenAI 当前快速开始提供原生 Windows 安装器，但 Codex 仓库的系统要求仍将 Windows 11 + WSL2 列为正式环境。需要 Linux 工具链时，优先选择 WSL2。

以管理员身份打开 PowerShell：

wsl --install

按提示重启电脑，打开 Ubuntu 并完成用户名/密码初始化，然后在 WSL 终端执行：

sudo apt update sudo apt install -y curl curl -fsSL https://chatgpt.com/codex/install.sh | sh codex --version

Microsoft 的一键 wsl --install 要求 Windows 10 2004（Build 19041）及以上或 Windows 11。WSL 与原生 Windows 使用不同的用户目录，.codex 配置也要放在你实际运行 Codex 的那一侧。

方案 C：npm（需要 Node.js）

前往 Node.js 官方下载页，下载并安装 LTS 版本。安装程序保持默认选项，确保同时安装 npm 并加入 PATH。重新打开 PowerShell后执行：

node --version npm --version npm install -g @openai/codex codex --version
03

macOS 安装步骤

官方仓库当前列出的系统要求为 macOS 12+，支持 Apple Silicon 和 Intel Mac。

方案 A：官方独立安装器（推荐、无需 Node.js）

curl -fsSL https://chatgpt.com/codex/install.sh | sh codex --version

方案 B：Homebrew

如果尚未安装 Homebrew，先使用其官网提供的命令：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

按照安装器最后给出的提示把 brew 加入 PATH，然后安装 Codex：

brew install --cask codex codex --version

方案 C：npm（需要 Node.js）

可以从 Node.js 官网安装 LTS 的 macOS .pkg，也可以在已有 Homebrew 的情况下执行：

brew install node node --version npm --version npm install -g @openai/codex codex --version

如果 npm 出现 EACCES 权限错误，不要直接使用 sudo npm。建议改用官方独立安装器，或使用 Node 版本管理器后再安装。

04

Linux 安装步骤

官方仓库当前列出的基线为 Ubuntu 20.04+ 或 Debian 10+。其他现代发行版通常也能运行，但需要自行保证 curl、证书和常用开发工具可用。

方案 A：官方独立安装器（推荐、无需 Node.js）

如果系统没有 curl，先按发行版安装：

# Ubuntu / Debian sudo apt update && sudo apt install -y curl # Fedora / RHEL sudo dnf install -y curl # Arch Linux sudo pacman -S curl

安装 Codex：

curl -fsSL https://chatgpt.com/codex/install.sh | sh codex --version

方案 B：npm（需要 Node.js/npm）

不建议依赖发行版仓库中的旧 Node.js。下面使用 Node.js 官方下载页当前推荐的 nvm 路线安装 LTS：

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.5/install.sh | bash # 关闭并重新打开终端后继续 nvm install --lts nvm use --lts node --version npm --version npm install -g @openai/codex codex --version

当前 @openai/codex npm 包声明 Node.js 16+；建议始终使用仍在维护的 LTS，而不是只满足最低版本。

05

验证安装与 定位命令

macOS / Linux / WSL

codex --version command -v codex type -a codex

Windows PowerShell

codex --version Get-Command codex -All where.exe codex

如果列出多个 codex 路径，说明存在重复安装。保留一种安装方式并清理旧路径，否则升级后仍可能启动旧版本。

06

获取 API 密钥

登录本站后前往 「令牌管理」，创建并复制一个 API 密钥。密钥只应保存在你自己的设备中，不要发给他人或提交到 Git 仓库。

07

创建并备份 .codex 配置目录

macOS / Linux / WSL

mkdir -p ~/.codex chmod 700 ~/.codex cp ~/.codex/auth.json ~/.codex/auth.json.bak 2>/dev/null || true cp ~/.codex/config.toml ~/.codex/config.toml.bak 2>/dev/null || true

Windows PowerShell

New-Item -ItemType Directory -Force "$HOME\.codex" Copy-Item "$HOME\.codex\auth.json" "$HOME\.codex\auth.json.bak" -ErrorAction SilentlyContinue Copy-Item "$HOME\.codex\config.toml" "$HOME\.codex\config.toml.bak" -ErrorAction SilentlyContinue
08

创建 auth.json

macOS / Linux / WSL

编辑 ~/.codex/auth.json，保存后执行 chmod 600 ~/.codex/auth.json。

Windows

使用记事本打开 $HOME\.codex\auth.json。保存时选择“所有文件”，避免文件名变成 auth.json.txt。

{ "OPENAI_API_KEY": "YOUR_API_KEY" }

把 YOUR_API_KEY 替换为上一步创建的密钥。JSON 最后一项后面不要添加逗号。

09

创建 config.toml

macOS / Linux / WSL

文件路径：~/.codex/config.toml

Windows

文件路径：$HOME\.codex\config.toml。确认扩展名不是 .toml.txt。

model_provider = "OpenAIGPT" model = "gpt-5.5" model_reasoning_effort = "xhigh" disable_response_storage = true preferred_auth_method = "apikey" [model_providers.OpenAIGPT] name = "OpenAIGPT" base_url = "https://tzhapi.cc.cd/v1" wire_api = "responses"

请保持 TOML 的引号、大小写和层级不变。若启动时提示模型不可用，请在本站控制台确认账号可用模型后再修改 model。

10

开始使用与 更新 Codex

进入你的项目目录后启动 Codex：

cd your-project-folder codex

按最初选择的安装方式更新，不要混用：

# macOS / Linux 官方独立安装器：重新运行 curl -fsSL https://chatgpt.com/codex/install.sh | sh # Windows 官方独立安装器：重新运行 powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex" # npm npm install -g @openai/codex@latest # Homebrew brew upgrade --cask codex

更新后重新打开终端，并运行 codex --version 确认。

11

常见问题 快速排查

codex: command not found

重新打开终端；检查 command -v codex、Get-Command codex；确认安装目录已加入 PATH。

npm 权限错误 / EACCES

不要使用 sudo npm。改用官方独立安装器，或使用 nvm 管理用户级 Node.js。

Windows 配置不生效

确认文件不是 auth.json.txt/config.toml.txt，并确认你运行的是原生 Windows 还是 WSL 中的 Codex。

401 / Unauthorized

检查 API 密钥是否完整、auth.json 是否为有效 JSON，以及令牌是否已启用且仍有额度。

更新后还是旧版本

系统中通常存在多个 codex。用 type -a codex 或 Get-Command codex -All 找到并清理旧安装。

WSL 项目速度较慢

建议把项目放在 WSL 的 ~/projects 中，而不是 /mnt/c 下，再从 WSL 终端运行 Codex。
