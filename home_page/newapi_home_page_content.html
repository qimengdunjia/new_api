Codex 快速配置

用 OpenAI 兼容接口运行 Codex

只需要准备两个文件：一个写模型与接口地址，一个写 API Key。下面的示例可直接作为入门配置参考。

~/.codex/config.toml
model_provider = "OpenAI"
model = "gpt-5.5"
review_model = "gpt-5.5"
model_reasoning_effort = "xhigh"
network_access = "enabled"

三步完成配置

1

创建配置目录

确认本机存在 ~/.codex 目录，用来保存 Codex 的配置文件与鉴权文件。

2

写入模型配置

在 config.toml 中指定模型、推理强度、接口地址和 Responses API 通信方式。

3

写入密钥并运行

在 auth.json 中放入 API Key，然后启动 Codex。真实 Key 不要提交到仓库。

推荐配置示例

~/.codex/config.toml
# 模型供应商名称 model_provider = "OpenAI" # 默认对话模型 model = "gpt-5.5" # 代码审查模型 review_model = "gpt-5.5" # 推理强度 model_reasoning_effort = "xhigh" # 不保存响应内容 disable_response_storage = true # 允许 Codex 访问网络 network_access = "enabled" # WSL 环境确认 windows_wsl_setup_acknowledged = true [model_providers.OpenAI] name = "OpenAI" base_url = "https://new-api-latest-xhh3.onrender.com/v1" wire_api = "responses" supports_websockets = true requires_openai_auth = true [features] responses_websockets_v2 = true
~/.codex/auth.json
{
  "OPENAI_API_KEY": "sk-替换为你的真实密钥"
}

安全提示：网页中只展示占位符。真实 OPENAI_API_KEY 只放在本机 ~/.codex/auth.json，不要写进 HTML、截图或 Git 提交。

检查要点

  • 接口地址base_url 指向 OpenAI 兼容服务的 /v1 地址。
  • 通信协议wire_api = "responses" 表示使用 Responses API。
  • 模型选择model 与 review_model 都使用 gpt-5.5。
  • 网络权限network_access = "enabled" 允许 Codex 访问网络。
